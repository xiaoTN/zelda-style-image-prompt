# 塞尔达风格图片提示词

用这个技能写稳定、简洁、可直接交给生图模型的塞尔达风格图片提示词。

## 自带依赖

写提示词前先读 `references/zelda-visual-style.md`，拿完整视觉规范、HUD 规则、三段式 UI 规则和元素池。

这个技能是独立提示词技能，随 skill 一起发布的 `references/` 已包含所需视觉规范。

## 提示词模板

默认使用这个模板。替换方括号里的内容，并保持行序。

```text
塞尔达原版游戏实况截图复刻
画风：任天堂 Switch 旷野之息/王国之泪实机游戏截图风格；不要电影 CG、不要写实摄影、不要 HDR 真实光影、不要湿润真实材质
TOTK 王国之泪默认造型，林克 + 塞尔达同行

海拉鲁地图区块：[地点或主题，例如「北京·故宫」/「林芝·墨脱雨林」/「古代遗迹工坊」]
画面要素：[3-5 个可识别关键词，只列名词或短词组]
玩法时刻：[一句话叙事，写角色正在做什么；体力动作写「正在消耗体力，显示耐力轮」，非体力动作写「不消耗体力，不显示耐力轮」]

塞尔达元素：[3-7 个关键词，只列名称不展开外观]
地名 UI 文字（区域发现）：顶部「发现！」 / 中间「[真实主地名或场景名]」 / 底部「[真实上级区域名或主题归属]」；按原版区域发现三段式呈现，底部上级区域右对齐、只在文字左侧带短横线、淡色发光、无矩形边框、无深色底纹
HUD：按 BOTW/TOTK 原版游戏实况自然出现，不堆满
文字：所有可读 UI 使用中文，按键动词只用中文，如「调查」「打开」「拾取」「烹饪」
```

## 填写规则

- 提示词控制在 10-13 行左右。不要加前景/中景/远景分层、镜头焦段、材质描述或长篇光线描述。
- `画风` 行不要再加 `cel-shading 扁平卡通色块、强黑描边轮廓、柔光绘画感、淡雅手绘风色调`。
- 默认使用 `TOTK 王国之泪默认造型`。只有用户明确要 BOTW，或场景明显适合 BOTW 时，才换成 `BOTW 旷野之息默认造型`。
- 尊重用户的角色边界。用户说不要林克，就去掉 `林克 +` 并写塞尔达单主角；用户只要林克，就移除塞尔达。
- 真实地点的所有可读 UI 地名必须是真实中文名。不要把 `海拉鲁`、`格鲁德高地`、`卡卡里科村` 等游戏内地名写进 UI。
- `海拉鲁地图区块` 是风格锚点字段名；即使用于真实地点，也不代表 UI 文字要写海拉鲁地名。
- `画面要素` 承担主体辨识度，使用地标名词、物体特征、轮廓、动作或产品细节等短关键词。
- `塞尔达元素` 只写紧凑列表，例如 `TOTK 左纳乌神庙、原版宝箱、希卡族研究员、波克布林营地、料理锅、远空飞行魔物`。不要展开形状、颜色、材质或位置。
- 如果场景没有真实上级区域，底部 UI 用简短中文主题归属，例如 `古代遗迹研究区` 或 `海边驿站外围`；不要写宣传语。

## 输出模式

只返回最终提示词代码块，不要附带任何其他说明。
